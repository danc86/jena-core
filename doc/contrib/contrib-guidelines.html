guidelines for contributors

request for listing

In your request for listing message to jena-devel, make sure you supply:

  • the name of your contribution
  • your name
  • a brief description of your contribution
  • the URL of your contribution's web-page or download

guidelines for contributions

Multiple-file contributions should be bundled into a single archive file, typically a .zip file. Pathnames in the file should be relative.

Java code

We recommend that Java code be bundled in the same style as Jena itself. So the zip should have a structure like this:

  • a README[.t[e]xt] which sumarises the contribution and links to any documentation.
  • a src directory containing the source code tree.
  • a lib directory containing any required jars. The contributions own jar can appear here.
  • a doc directory for any documentation.
  • an Ant buildfile (typically build.xml). Even if the user isn't expected to build from source, the buildfile can have a test target for running your contributions tests.
  • a clear statement of your contribution's licence.

ontologies

Your ontology should have a non-filename URI. If possible, that URL should be the URL from which the ontology content can be downloaded.
